<task type="auto">

  <files>src/tools/tickets.ts</files>
  <action>
Add two tools inside the existing `registerTicketTools` function, after `get_ticket`.

**create_ticket tool:**
- Name: `create_ticket`
- Description: "Create a new support ticket. Provide customer_id or customer_name to link the ticket."
- Params (Zod inline):
  - `customer_id`: z.string().uuid().optional()
  - `customer_name`: z.string().optional()
  - `subject`: z.string().min(1)
  - `description`: z.string().optional()
  - `priority`: TicketPriority.optional()

Implementation:
1. If neither customer_id nor customer_name provided, return error: "Either customer_id or customer_name is required"
2. If customer_id provided directly, use it (skip name resolution — ID takes precedence per Phase 2 decision)
3. If only customer_name provided, resolve using Phase 2 pattern:
   - Query `supabase.from('customers').select('id, name').ilike('name', '%${customer_name}%')`
   - 0 matches: return `No customer found matching "${customer_name}"`
   - Multiple matches: return JSON with error message and matches array (id + name for each)
   - 1 match: use matches[0].id as finalCustomerId
4. Validate customer exists: `supabase.from('customers').select('id').eq('id', finalCustomerId).single()`
   - If not found: return `Customer not found: ${finalCustomerId}`
5. Insert: `.insert({ customer_id: finalCustomerId, subject, description: description || null, priority: priority || 'medium' }).select()`
6. Error: return `Database error: ${error.message}`
7. Success: return `JSON.stringify(data[0], null, 2)`

**close_ticket tool:**
- Name: `close_ticket`
- Description: "Close a support ticket and mark it as resolved with optional resolution note"
- Params (Zod inline):
  - `id`: z.string().uuid("Ticket ID must be a valid UUID")
  - `resolution`: z.string().optional()

Implementation:
1. Fetch current ticket: `supabase.from('tickets').select('status, closed_at').eq('id', id).single()`
2. If not found: return `Ticket not found: ${id}`
3. If `existing.status === 'closed'`: return `Ticket ${id} is already closed (closed on ${existing.closed_at})`
4. Update: `.update({ status: 'closed', closed_at: new Date().toISOString(), resolution: resolution || null }).eq('id', id).select()`
5. Error: return `Database error: ${error.message}`
6. Success: return `JSON.stringify(data[0], null, 2)`
