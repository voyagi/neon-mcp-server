Tool registration:
- Name: `get_summary`
- Description: "Get a dashboard summary with customer counts, ticket stats, product catalog value, and recent activity"
- Params: empty object `{}`

Implementation — use Promise.all to run queries in parallel batches:

**Batch 1 — Customer counts by status (3 queries):**
```
supabase.from('customers').select('id', { count: 'exact', head: true }).eq('status', 'active')
supabase.from('customers').select('id', { count: 'exact', head: true }).eq('status', 'inactive')
supabase.from('customers').select('id', { count: 'exact', head: true }).eq('status', 'lead')
```

**Batch 2 — Ticket counts (6 queries):**
- Open tickets (status != 'closed')
- Closed tickets (status = 'closed')
- Open by priority: urgent, high, medium, low (each with .neq('status', 'closed'))

**Batch 3 — Products + Recent activity (3 queries):**
- All products: `select('price_cents, category')`
- Customers created in last 7 days: count with `.gte('created_at', sevenDaysAgo)`
- Tickets closed in last 7 days: count with `.eq('status', 'closed').gte('closed_at', sevenDaysAgo)`

Calculate `sevenDaysAgo` as: `new Date(Date.now() - 7 * 24 * 60 * 60 * 1000).toISOString()`

Run all queries via a single `Promise.all([...batch1, ...batch2, ...batch3])` and destructure results.

**Product aggregation:**
```
const totalValue = products.reduce((sum, p) => sum + p.price_cents, 0);
const categoryTotals: Record<string, number> = {};
for (const product of products) {
  categoryTotals[product.category] = (categoryTotals[product.category] || 0) + product.price_cents;
}
```

**Format price helper** — define locally (same pattern as products.ts):
```
function formatPrice(cents: number): string {
  return `$${(cents / 100).toFixed(2)}`;
}
```

**Response shape:**
```json
{
  "customers": {
    "active": N, "inactive": N, "leads": N, "total": N
  },
  "tickets": {
    "open": N, "closed": N, "total": N,
    "by_priority": { "urgent": N, "high": N, "medium": N, "low": N }
  },
  "products": {
    "total_value": "$X.XX",
    "by_category": [{ "category": "...", "value": "$X.XX" }]
  },
  "recent_activity": {
    "customers_created_this_week": N,
    "tickets_closed_this_week": N
  }
}
```

Use `?? 0` for all count values to handle null. For products query error, return partial summary with an error note for the products section rather than failing the entire tool.

Export: `export function registerAnalyticsTools(server: McpServer): void { ... }`
