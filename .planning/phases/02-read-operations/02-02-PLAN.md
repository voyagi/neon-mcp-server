**Step 1: Update ListTicketsSchema in validation.ts**

Add `customer_name` as an optional string field to `ListTicketsSchema`:
```typescript
export const ListTicketsSchema = z.object({
  status: TicketStatus.optional(),
  customer_id: z.string().uuid("Customer ID must be a valid UUID").optional(),
  customer_name: z.string().optional(),
  priority: TicketPriority.optional(),
});
```

This enables Claude to filter tickets by customer name (e.g., "show me tickets from John") without knowing the customer's UUID.

**Step 2: Create `src/tools/tickets.ts`**

Create with a `registerTicketTools(server: McpServer)` function that registers two MCP tools.

**list_tickets tool:**
- Tool name: `list_tickets`
- Description: "List tickets with optional filters by status, priority, customer ID, or customer name"
- Input schema: use TicketStatus, TicketPriority, and string fields inline in inputSchema (status optional, customer_id optional UUID, customer_name optional string, priority optional)
- **Customer name resolution** (two-step query):
  If `customer_name` is provided (and `customer_id` is NOT), first resolve the name:
  ```typescript
  const { data: matchingCustomers } = await supabase
    .from('customers')
    .select('id')
    .ilike('name', `%${customer_name}%`);
  ```
  If no matching customers found, return early with empty results and message "No customers match the name \"{customer_name}\""
  If matches found, extract the IDs and use `.in('customer_id', matchingCustomerIds)` on the main query
  If `customer_id` is provided, it takes precedence over `customer_name` (ignore customer_name)
- Main query: `supabase.from('tickets').select('*, customers(name)').order('created_at', { ascending: false })`
- Conditional filters:
  - If `status` provided: `.eq('status', status)`
  - If `customer_id` provided: `.eq('customer_id', customer_id)`
  - If customer IDs resolved from name: `.in('customer_id', resolvedIds)`
  - If `priority` provided: `.eq('priority', priority)`
- Post-process results: flatten `customers.name` to `customer_name` at the ticket level for cleaner output:
  ```typescript
  const results = data.map(ticket => ({
    ...ticket,
    customer_name: ticket.customers?.name || "Unknown Customer",
    customers: undefined, // remove nested object
  }));
  ```
- Response format:
  ```json
  {
    "results": [...tickets with customer_name flattened...],
    "count": N,
    "message": "No tickets match your filters" // only when count is 0
  }
  ```

**get_ticket tool:**
- Tool name: `get_ticket`
- Description: "Get ticket details including linked customer information"
- Input schema: id as UUID string
- Query with nested select:
  ```typescript
  supabase.from('tickets').select(`
    *,
    customers (
      id,
      name,
      email,
      company
    )
  `).eq('id', id).single()
  ```
- If error or null data, return "Ticket not found: {id}"
- Flatten the `customers` nested object to `customer` (singular) for cleaner output:
  ```typescript
  const response = {
    ...data,
    customer: data.customers ? {
      id: data.customers.id,
      name: data.customers.name,
      email: data.customers.email,
      company: data.customers.company,
    } : null,
    customers: undefined,
  };
  ```
- Handle null customer gracefully: if `data.customers` is null, set `customer: null` (not an error)
- Return as `{ content: [{ type: "text", text: JSON.stringify(response, null, 2) }] }`

**Imports needed:**
