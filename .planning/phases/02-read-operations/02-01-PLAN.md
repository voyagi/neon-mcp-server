Create `src/tools/customers.ts` with a `registerCustomerTools(server: McpServer)` function that registers two MCP tools.

**list_customers tool:**
- Tool name: `list_customers`
- Description: "List all customers, optionally filter by status or company name"
- Input schema: use `ListCustomersSchema` fields inline in the inputSchema object (status as optional enum, company as optional string). Refer to validation.ts for the Zod schemas but pass them directly to `server.tool()` inputSchema.
- Query: `supabase.from('customers').select('*')` with `.order('name', { ascending: true })`
- Conditional filters:
  - If `status` provided: `.eq('status', status)`
  - If `company` provided: `.ilike('company', '%${company}%')` for partial case-insensitive match
- Error handling: destructure `{ data, error }`, return error message if error
- Response format:
  ```json
  {
    "results": [...all customer objects...],
    "count": N,
    "message": "No customers match your filters" // only when count is 0
  }
  ```
- Return as `{ content: [{ type: "text", text: JSON.stringify(response, null, 2) }] }`

**get_customer tool:**
- Tool name: `get_customer`
- Description: "Get a single customer by ID, including their ticket summary"
- Input schema: use `GetCustomerSchema` field (id as UUID string)
- Three queries:
  1. Customer: `supabase.from('customers').select('*').eq('id', id).single()`
  2. Total and open ticket counts: Two count queries using `{ count: 'exact', head: true }`
     - Total: `.from('tickets').select('id', { count: 'exact', head: true }).eq('customer_id', id)`
     - Open: same but add `.neq('status', 'closed')`
  3. Recent tickets: `.from('tickets').select('subject, status, created_at').eq('customer_id', id).order('created_at', { ascending: false }).limit(3)`
- If customer query returns error or null data, return "Customer not found: {id}"
- Response format: spread customer fields plus `open_tickets_count`, `total_tickets_count`, `recent_tickets` array
- Return as `{ content: [{ type: "text", text: JSON.stringify(response, null, 2) }] }`

- `import { CustomerStatus } from "../lib/validation.js";`

The research shows `server.registerTool()` but verify the actual method name from the SDK (it may be `server.tool()`).
