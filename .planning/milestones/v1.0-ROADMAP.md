## Milestone Summary

**Key Decisions:**

- Supabase over raw Postgres (easier setup for prospects, free tier)
- Service role key without RLS (demo simplicity over production patterns)
- stdio transport only (standard for Claude Desktop, no HTTP complexity)
- Hardcoded schema resource over runtime introspection (reliability)
- Zod .strict() with custom enum errorMaps (helpful validation messages)
- Read-before-write phase ordering (validate patterns before destructive ops)
- Promise.all for analytics queries (minimize latency)
- Subquery FK resolution in seed data (portable, no hardcoded UUIDs)

**Issues Resolved:**

- README data inaccuracies (ticket counts, catalog value) caught by verification and fixed in gap closure plan 04-03
- Incomplete demo conversation tool coverage (6/7 types) caught by verification and closed with search_products exchange

**Issues Deferred:**

- src/lib/supabase.ts references non-existent seed/setup.md (info-level, pre-existing from scaffolding)

**Technical Debt Incurred:**

- No test framework (deliberate for portfolio demo — time better spent on polish)
- No pagination on list operations (demo scale doesn't need it)
