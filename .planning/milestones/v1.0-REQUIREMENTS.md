# Requirements Archive: v1.0 MVP

**Archived:** 2026-02-09
**Status:** All v1 requirements complete

## v1 Requirements

### Infrastructure

- [x] **INFR-01**: MCP server exposes database schema as resource at `schema://tables` listing all tables and columns — v1.0
- [x] **INFR-02**: All tool inputs validated with Zod schemas; invalid inputs return helpful error messages explaining what's wrong and how to fix it — v1.0
- [x] **INFR-03**: All logging uses stderr only; no stdout writes that could corrupt the MCP JSON-RPC protocol stream — v1.0

### Customers

- [x] **CUST-01**: `list_customers` tool returns all customers with optional filters by status (active/inactive/lead) and company name — v1.0
- [x] **CUST-02**: `get_customer` tool returns a single customer by UUID with all fields — v1.0
- [x] **CUST-03**: `create_customer` tool adds a new customer with name, email, company, and status; returns created record — v1.0
- [x] **CUST-04**: `update_customer` tool modifies customer fields by UUID; returns updated record — v1.0

### Tickets

- [x] **TICK-01**: `list_tickets` tool returns tickets with optional filters by status (open/in_progress/closed), customer ID, and priority — v1.0
- [x] **TICK-02**: `get_ticket` tool returns ticket details including linked customer info via JOIN (not just customer_id) — v1.0
- [x] **TICK-03**: `create_ticket` tool creates a support ticket linked to existing customer by UUID; validates customer exists — v1.0
- [x] **TICK-04**: `close_ticket` tool marks ticket as resolved with closed_at timestamp; validates ticket is not already closed — v1.0

### Products

- [x] **PROD-01**: `list_products` tool returns all products with name, category, price, and description — v1.0
- [x] **PROD-02**: `search_products` tool finds products by name or category using case-insensitive matching — v1.0

### Analytics

- [x] **ANLT-01**: `get_summary` tool returns composite dashboard stats: total customers by status, open vs closed ticket counts, total product catalog value — v1.0

### Seed Data

- [x] **SEED-01**: Seed SQL contains 22 customers (diverse companies, mixed statuses), 32 tickets (varied subjects, priorities, open/closed mix with timestamps), 12 products (realistic SaaS names, categories, pricing) — v1.0
- [x] **SEED-02**: Setup guide documents how to create Supabase project, create tables, and run seed SQL — v1.0

### Polish

- [x] **PLSH-01**: README includes project overview, feature list, demo conversation path, and self-serve setup guide for prospects — v1.0
- [x] **PLSH-02**: Claude Desktop integration config example with copy-paste JSON for `claude_desktop_config.json` — v1.0
- [x] **PLSH-03**: Demo conversation script covering all 7 tool types: list, filter, create, close, search, analytics, schema introspection — v1.0
